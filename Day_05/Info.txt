//***************** __ updateOne / updateMany __ ********************
If usert=true and the updated doc not found, it will be created. The fields inside the new doc  will be MERGED from filter and update stage. In case there is a duplication fields, update stage values will have a priority over the filter.

Used for:
    updateOne = Update first document (even found some). First means the first by index. Atomic operation.
                If changed field exists - it will be updated with new value. If not - it will be created.
    updateMany = Update all docs that are align by filtering criterias.

Schema:
    db.collection.updateOne(filter, update, options)

Example-0 (updateOne + upsert):
    Empty DB, command:
        db.products.updateOne(
            { "sku": "A123", "status": "active", "secretField": "invisible" },      // filter
            { "$set": { "quantity": 10, "status": "discontinued" } },               // update
            { "upsert": true }                                                      // options
        );
    
    Now DB has:
        {
            "_id": ObjectId("..."),         // autocreated
            "sku": "A123",                  // taken from update section
            "secretField": "invisible",     // taken from filter section
            "status": "discontinued",       // update value has priority over filter
            "quantity": 10
        }

Example-1 (updateMany):
    db.users.updateMany(
        { status: "inactive" }, 
        { $set: { archived: true } })

    Returned:
    {
        "acknowledged": true, 
        "matchedCount": 5,      // number of found by filter criteria
        "modifiedCount": 3,     // number of modified
        "upsertedId": null,     
        "upsertedCount": 1    
    }

Example-2:
    db.players.updateOne(
        { "nickname": "DragonSlayer77" },                   // Filter of records to be updated
        {                                                   // What and how to change inside the founded doc
            $set: {                                             // set new values according to the fields. If not found - field will be created
                "status": "VIP", 
                "lastActivity.type": "purchase" 
            },
            $unset: { "reminders.promotionNotified": "" },      // remove field "promotionNotified" from object "reminders". For array elments it will set value "null".
            $inc: {
                "wallet.gold": 500,                             // Add to "gold" inside "wallet" 500 points (wallet.gold += 500)
                "stats.totalPurchases": 1                       // Add to "totalPurchases" inside "stats" 1 point (stats.totalPurchases += 1)
                "slotRemaining": -1                             // slotRemaining -= 1 
            },
            $mul: { "multipliers.exp": 1.2 },                   // "exp" inside "multipliers" *= 1.2
            $max: { "stats.highScore": 1500 },                  // Update highestScore value to 1500 only if current value is less than 1500, otherwise stay as is
            $min: { lowestScore: 100 },                         // Update lowestScore value to 100 only if current value is more than 100, otherwise stay as is
            $addToSet: { "achievements": "First Purchase" },    // Add element to the array of "achievements" only if it yet not there
            $pull: { "activeEffects": { "expired": true } },    // Delete all elements from array "activeEffects" that aligned by criteria
            $pop: { "avatars" : 1 }                             // Delete one element - first for (-1) or last for (1) - from array "avatars"
            $push: { "moto": "We are the champs!!!!"}           // Add element to the end of array "moto". If no such array - it will be created.
            $currentDate: { "lastActivity.date": true },        // Set current date into the field (obj "lastActivity", field "date"). Value should always be "true"
            $setOnInsert: { "createdAt": new Date() }           // Set field data ONLY if the doc was created from scratch (means, with upsert)
            $rename: {"oldValue": "newValue"}                   // Rename field from "oldValue" to "newValue". Used set/unset as atomic operation inside Update           
        },
        {                                                   // Options
            upsert: true                                        // flag to create doc if not found
            writeConcern: { w: "majority", j: true },
            collation: { locale: "en", strength: 2 },           // Language conditions
            hint: { "email": 1 }                                // Force to use index by email
        } 
    )

    Hook (trick):
        You can use as secoond arg an array: db.collection.updateOne(filter, [change1, change2,...], options). Difference between this and regular schema is that here Mongo will do changes step-by-step, once in regular it will do in parallel. Due to this parallelism you can't do more than 1 operation over single field in single update, but with array you can. Inside the array you'll need to use aggregation commands only.

    Returned:
        {
            "acknowledged": true,
            "matchedCount": 0,                                      // doc not found
            "modifiedCount": 0,                                     // num of modified docs
            "upsertedId": ObjectId("65f1a2b3c4d5e6f7a8b9c0d1"),     // _id of new created doc
            "upsertedCount": 1                                      //num of created docs
        }


//***************** __ $ operator __ ********************

    Used:
        update data in first element(always the first, even found more than 1) of array matches to search condition

    How it works:
        in search section Mongo look for an element with specific criteria and remeber the index of found element. In update section Mongo substitude operator $ by remembered index. For multi change situation $[<name>] is a placeholder, and <name> could be any meaningfull value. Main condition - the same name should be used in arrayFilters line. If you do not provide the name (means $[] stay empty) it will update all the elements in the array, without any condition. Line of arrayFilters provide a conditions to detect elements to update. To allow working with array elements we need to do a search inside the array in querry (even we know to filter the exactly doc, but inside the doc in array we need to search for an eelment to get it possition).

    Example - single change:
        db.collection.updateOne(
            { "tags": "old_value" },                     // search criteria
            { "$set": { "tags.$": "new_value" } }        // update using $ operator
        )

    Example - multiply change:
        Document:
            { _id: 1, 
            scores: [ 
                { val: 40 }, 
                { val: 55 }, 
                { val: 90 } 
            ] }

        Command:
            db.col.updateOne(
                { _id: 1 },                                                     // filter section
                { $set: { "scores.$[currentElement].failed": true } },          // how exactly we should update  
                { arrayFilters: [ { "currentElement.val": { $lt: 60 } } ] }     // what exactly we should update 
            )

        Returned:
            { _id: 1, 
            scores: [ 
                { val: 40, failed: true }, 
                { val: 55, failed: true }, 
                { val: 90 } 
            ] }


    Example - total change:
        db.students.updateOne(
            { _id: 1 },                                     // select doc to update
            { $inc: { "grades.$[]": 10 } }                  // for any element of array "grades" add 10 points
        )

    Example - update:
        Document:
            { _id: 1, 
            scores: [ 
                { id: "a", val: 50 }, 
                { id: "b", val: 80 } 
            ] }

        Command:
            db.col.updateOne(
                { "scores.id": "b" },                   // filter sections, search only the element that has scores[i].id = b
                { $set: { "scores.$.val": 99 } }        // update section, set into the found element scores[i].val = 99
            )

        Result:
             { _id: 1, 
                scores: [ 
                    { id: "a", val: 50 }, 
                    { id: "b", val: 99 } 
            ] } 

    Example - find:
        Document:
            { _id: 1, 
            scores: [ 
                { id: "a", val: 50 }, 
                { id: "b", val: 80 } 
            ] }

        Command:
            db.col.find(
                { "scores.val": { $gte: 70 } },     // filter stage, first element with score.value >= 70 should be detected 
                { "scores.$": 1 }                   // projection stage (which fields to return), as this is find command
            )

        Result:
             { _id: 1,                              // returned as by default it's 1
                scores: [ 
                    { id: "b", val: 80 }            // returned as we asked for the item having score.value >= 70 
            ] } 

    Example:
        Command:
            db.col.updateOne(
                { _id: 10, "scores.id": "b" },          // what to update stage - we know the document, but search inside the array
                { $set: { "scores.$.val": 12 } }        // how to update stage - we can use $ as placeholder for substitution 
            )

        Result:
             { _id: 10, 
                scores: [ 
                    { id: "a", val: 50 },               
                    { id: "b", val: 12 } 
            ] } 

ATTENTION
    1) You can't do 2 or more operations on the same field in same request (for example, set & inc) = error "Conflicting update operators"
    2) You can't do operation on parent object and any operation on it's children
    3) Use $pull instead of $unset while working with arrays
    4) Mongo can modify the doc in other order than the fields appear in the request, but operation still remains atomic = all will be updated OR nothing will be updated 
    5) Inside $set you can't use the other fields of the document, it's expect the defined values

//***************** __ replaceOne __ ********************
Used for:
    Totally replace old document with a new one. _id of the doc stay the same!!! In case the doc does not exist, but upsert stay as true, the new doc will be created. Put attention: the fields in new created doc will be taken from upsert section only!

Schema:
    db.collection.replaceOne(
        { "key": "value" },             // filter to search doc
        { "newKey": "val" },            // replacement object (new document that will replace an old one)
        {                               // options
            upsert: <boolean>,
            writeConcern: <document>,
            collation: <document>,
            hint: <document|string>
        }
    )

Example-1:
    Empty DB, command
        db.products.replaceOne(
            { "sku": "A123", "status": "active", "email":"asd@zxc.com" },       // filtering stage
            { "sku": "A123", "status": "discontinued", "quantity": 10 },        // upsert stage
            { "upsert": true }
        );
    
    In DB now:
        [
            {
                "_id": ObjectId("65f1a2b3c4d5e6f7a8b9c0d1"),                    // created automatically
                "sku": "A123",                                                  // merged from upsert stage
                "status": "discontinued",                                       // merged from upsert stage
                "quantity": 10                                                  // merged from upsert stage
            }
        ]

Example-2:
    db.backups.replaceOne(
        { "targetId": 50 },                                             // filter
        { "targetId": 50, "data": "new_blob", "updatedAt": "2026" }     // full new object
    )

Returned:
    {
        "acknowledged": true,
        "matchedCount": 1,
        "modifiedCount": 1,
        "upsertedId": null,
        "upsertedCount": 0
    }

//***************** __ deleteOne / deleteMany __ ********************
Used for:
    Delete all docs aligned with search filter

Example-1:
    db.alerts.deleteMany(
        { "level": "info", "isRead": true },    //Filter
        { writeConcern: { w: 1 } }              //Options
    )

Returned:
    {
        "acknowledged": true,
        "deletedCount": 142         //number of deleted docs
    }
